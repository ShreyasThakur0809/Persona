# Persona

**ZK proof-of-humanity protocol on Monad blockchain.**

Persona lets users prove they are a real, unique human over a minimum age — without revealing their identity. It uses Groth16 zero-knowledge proofs (Circom + snarkjs) verified on-chain by a Solidity verifier, issuing a non-transferable Soulbound Token (SBT) as proof of humanity.

---

## Architecture

```
packages/
  circuits/   — Circom age proof circuit + snarkjs ceremony artifacts
  contracts/  — Hardhat + Solidity (Groth16Verifier, IdentityRegistry, PersonaToken)
  webapp/     — Next.js 14 frontend (wagmi v2, RainbowKit v2, Tailwind)
```

### ZK Circuit (`ageProof.circom`)

Proves, without revealing personal data:
- The user is over a minimum age (full date-aware check)
- A country code commitment (countryHash)
- A unique nullifier to prevent double-registration

**Public signals layout (8 signals):**

| Index | Signal | Description |
|-------|--------|-------------|
| 0 | `isOver18` | 1 if age ≥ minAge, else 0 |
| 1 | `nullifier` | Poseidon(secret) — prevents re-registration after revoke |
| 2 | `countryHash` | Poseidon(countryCode) |
| 3 | `uniquenessHash` | Poseidon(idNumber, secret) |
| 4 | `currentYear` | Public input |
| 5 | `currentMonth` | Public input |
| 6 | `currentDay` | Public input |
| 7 | `minAge` | Public input |

### Contracts

| Contract | Description |
|----------|-------------|
| `Groth16Verifier` | Real BN254 verifier generated by snarkjs ceremony |
| `IdentityRegistry` | Core: verify ZK proof, store identity flags, manage nullifiers |
| `PersonaToken` | SBT (ERC-721 with transfers disabled) minted on successful verification |
| `MockGroth16Verifier` | Always-true verifier used in tests |

## Getting Started

### Prerequisites

- Node.js 18+, pnpm
- circom 2.2.3 (place binary at `~/bin/circom`)

### Install

```bash
pnpm install
```

### Run tests

```bash
cd packages/contracts
pnpm test        # 57 tests, uses MockGroth16Verifier
```

### Start frontend

```bash
cd packages/webapp
pnpm dev         # http://localhost:3000
```

### Deploy to Monad testnet

```bash
cd packages/contracts
pnpm deploy:testnet
```

### Recompile circuits (requires circom)

```bash
cd packages/circuits
PATH="$HOME/bin:$PATH" bash scripts/compile.sh
```

## How It Works

1. User connects wallet on the webapp.
2. User provides ID document data (birthdate, country, ID number, secret).
3. The browser generates a Groth16 ZK proof using snarkjs (WASM circuit).
4. The proof and public signals are submitted to `IdentityRegistry.verify()` on Monad.
5. The contract verifies the proof on-chain, checks the nullifier is unused, and mints a `PersonaToken` SBT to the user's address.

The nullifier is stored permanently — revoking a token does not clear it, preventing re-registration.

## dApp Integration

Implement `IPersonaRegistry` to gate access:

```solidity
import "./interfaces/IPersonaRegistry.sol";

IPersonaRegistry registry = IPersonaRegistry(IDENTITY_REGISTRY_ADDRESS);

function onlyHuman(address user) internal view {
    require(registry.isVerified(user), "Not verified");
}
```

## Notes

- `evmVersion: "cancun"` is required — OpenZeppelin 5.x uses the `mcopy` opcode.
- Tests bypass real ZK proof generation via `MockGroth16Verifier`.
- Set `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` for production WalletConnect support.
